# v0.1 Playable Onboarding — Draft Pool Completion

## Overview

Complete the draft pool flow so a player can select 1-3 wrestlers from a pre-generated pool of 6, guided by scout reports instead of raw stats, and land on a dashboard showing their roster.

## Goals

- Eliminate LLM calls during the draft flow (instant page loads)
- Allow multi-select (1-3 wrestlers) instead of single pick
- Hide raw stats and potential from the player; use scout reports instead
- Show drafted roster on the dashboard

- Account creation via allauth
- Heya name selection (`setup_heya_name` view, `ShikonaService`)
- Draft pool generation (`DraftPoolService.generate_draft_pool()`) — but calls LLM at request time
- Single wrestler draft selection (`_handle_draft_selection`) — picks 1 wrestler
- Redirect guards (decorators: `heya_required`, `setup_in_progress`, `draft_pending`)

## Design

### 1. Pre-generated Wrestler Pool Model

A new model `DraftPoolEntry` stores pre-generated wrestlers ready to be drafted. Each entry contains all the data needed to create a `Rikishi` on selection, plus a scout report.

```
DraftPoolEntry
  - shikona_name: str (Japanese kanji)
  - shikona_transliteration: str (romaji)
  - shikona_interpretation: str (meaning)
  - shusshin_country_code: str
  - shusshin_jp_prefecture: str
  - shusshin_display: str
  - potential: int (hidden from player)
  - strength: int (hidden from player)
  - technique: int (hidden from player)
  - balance: int (hidden from player)
  - endurance: int (hidden from player)
  - mental: int (hidden from player)
  - scout_report: str (player-visible narrative)
  - is_available: bool (default True, set to False when drafted)
  - created_at: datetime (auto)
```

No FK relationships — this is a flat staging table. When a player drafts a wrestler, the entry is used to create a proper `Rikishi` + `Shikona` + `Shusshin`, then marked unavailable.

### 2. Scout Report Generation

Each `DraftPoolEntry` gets a scout report generated by LLM at pool creation time (not at request time). The report is a 2-3 sentence narrative that hints at the wrestler's strengths and weaknesses without revealing exact stats or potential.

The prompt receives the wrestler's stats, potential, origin, and shikona, and produces a scouting paragraph. Example output:

> "A powerfully built wrestler from Osaka who relies on his low center of gravity. Observers note his impressive grip strength, though some question whether his footwork can keep up at the highest level."

The scout report is stored in the `scout_report` field and served directly to the template.

### 3. Management Command: `seed_draft_pool`

A Django management command `seed_draft_pool` generates entries:

```bash
uv run python manage.py seed_draft_pool --count 30
```

This command:
1. Uses `RikishiGenerator` to generate wrestlers (stats, shikona via OpenAI, shusshin)
2. For each wrestler, calls OpenAI to generate a scout report
3. Stores each as a `DraftPoolEntry` with `is_available=True`
4. Skips duplicates (by shikona name) against existing pool entries and `Shikona` table
5. Logs progress and summary

This runs offline/ahead of time, so LLM latency is not user-facing.

### 4. Updated Draft Pool View

The `setup_draft_pool` view changes:

**GET:**
- Draws 6 available `DraftPoolEntry` records (random order) instead of generating on the fly
- If fewer than 6 available, logs a warning (pool needs replenishing)
- Stores the 6 entry IDs in session for POST validation
- Passes entries to template (shikona, origin, scout report only — no stats)

**POST:**
- Accepts a list of 1-3 selected entry IDs (multi-select instead of single radio)
- Validates: at least 1, at most 3, all IDs are in the session's 6
- For each selected entry: creates `Shikona`, `Shusshin`, `Rikishi` (using existing `create_rikishi_from_selection` logic)
- Marks selected entries as `is_available=False`
- Redirects to dashboard

### 5. Updated Draft Pool Template

Replace the current template that shows raw stats with:

- 6 wrestler cards in a responsive grid (keep existing layout structure)
- Each card shows: shikona (kanji + romaji + interpretation), origin, scout report
- No stats, no potential tier, no progress bars
- Checkbox selection (multi-select) instead of radio buttons (single select)
- Visual feedback: selected cards get a highlighted border (keep existing `peer-checked` pattern)
- Counter showing "X/3 selected" that updates as player toggles
- "Confirm Roster" button enabled when 1-3 wrestlers are selected
- Validation: disable further selection when 3 are checked

### 6. Updated Dashboard

Add a "Your Roster" section below the existing welcome hero:

- List of drafted wrestlers as cards
- Each card shows: shikona (kanji + romaji), origin, scout report
- No stats visible (consistent with draft — player never sees raw numbers)

### 7. Updated Heya Name Selection

The `setup_heya_name` view currently calls `ShikonaService.generate_shikona_options()` which calls OpenAI at request time. This should also use pre-generated data.

Add a `ShikonaPoolEntry` model (or extend the seed command) to pre-generate heya name options. The `setup_heya_name` view draws 3 from the pool instead of generating live.

```
ShikonaPoolEntry
  - name: str (Japanese kanji)
  - transliteration: str (romaji)
  - interpretation: str (meaning)
  - pool_type: str ("heya" or "rikishi")
  - is_available: bool (default True)
  - created_at: datetime (auto)
```

The `seed_draft_pool` command also seeds heya shikona (or a separate `seed_shikona_pool` command).
